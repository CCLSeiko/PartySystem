# DonationSystem 需求文件

## 系統概述

DonationSystem 是一個線上捐款平台，支援多種捐款方式（現金、信用卡扣款、郵政劃撥），提供使用者便捷的捐款體驗與後台管理功能。

## 目標使用者

1. **捐款者**：一般民眾，希望透過線上平台進行捐款
2. **管理員**：機構管理人員，負責管理捐款紀錄與統計
3. **系統管理員**：技術人員，負責系統維護與設定

## 功能需求

### 1. 使用者管理

#### 1.1 註冊
- 使用者可透過電子郵件註冊帳號
- 需驗證電子郵件地址
- 密碼需符合安全性要求（至少 8 字元，包含大小寫與數字）

#### 1.2 登入
- 使用者可使用電子郵件與密碼登入
- 支援記住我功能（長期 Session）
- 登入失敗次數限制（防止暴力破解）

#### 1.3 個人資料管理
- 使用者可查看與編輯個人資料（姓名、電話、地址等）
- 使用者可更改密碼
- 使用者可查看捐款歷史紀錄

### 2. 捐款管理

#### 2.1 建立捐款
- 使用者可選擇捐款方式（現金、信用卡、郵政劃撥）
- 使用者可輸入捐款金額
- 使用者可選擇捐款用途（一般捐款、專案捐款等）
- 使用者可填寫備註資訊
- 系統需驗證捐款金額（最小金額限制）

#### 2.2 查詢捐款
- 使用者可查看自己的捐款紀錄
- 使用者可搜尋特定捐款（依日期、金額、用途等）
- 使用者可下載捐款收據（PDF 格式）

#### 2.3 取消捐款
- 使用者可在一定時間內取消捐款（例如 24 小時內）
- 取消捐款需記錄原因

### 3. 支付整合

#### 3.1 信用卡扣款
- 整合第三方支付閘道（如 Stripe、藍新金流）
- 支援主流信用卡（Visa、MasterCard、JCB 等）
- 支援 3D Secure 驗證
- 支援分期付款（可選）

#### 3.2 郵政劃撥
- 產生郵政劃撥單（含劃撥帳號、金額、備註）
- 提供劃撥單下載與列印功能
- 記錄劃撥單狀態（已產生、已寄出、已收款）

#### 3.3 現金捐款
- 記錄現金捐款資訊（捐款人、金額、時間、地點）
- 提供收據列印功能

### 4. 管理後台

#### 4.1 捐款管理
- 查看所有捐款紀錄
- 搜尋與篩選捐款（依日期、金額、捐款方式、狀態等）
- 匯出捐款資料（CSV、Excel 格式）

#### 4.2 統計報表
- 捐款總額統計（日、週、月、年）
- 捐款方式分佈統計
- 捐款用途分佈統計
- 捐款者統計（新捐款者、回流捐款者）

#### 4.3 使用者管理
- 查看使用者列表
- 搜尋使用者（依姓名、電子郵件等）
- 停用/啟用使用者帳號

#### 4.4 系統設定
- 設定捐款最小金額
- 設定捐款用途選項
- 設定支付閘道參數

## 非功能需求

### 1. 效能需求
- 系統回應時間：頁面載入 < 3 秒，API 回應 < 1 秒
- 支援同時 1000 人使用
- 資料庫查詢效能：95% 查詢 < 500ms

### 2. 安全性需求
- 使用 HTTPS 傳輸資料
- 密碼加密儲存（使用 bcrypt 或 Argon2）
- 支付資訊加密儲存
- 防止 SQL 注入、XSS、CSRF 攻擊
- 定期安全漏洞掃描

### 3. 可用性需求
- 系統可用性：99.9%（每月 downtime < 43 分鐘）
- 支援多瀏覽器（Chrome、Firefox、Safari、Edge）
- 支援行動裝置（RWD 設計）

### 4. 可擴充性需求
- 模組化設計，易於新增功能
- 支援水平擴充（負載均衡）
- API 設計符合 RESTful 標準

### 5. 法規遵循
- 符合個資法（個人資料保護）
- 符合稅務規定（捐款收據格式）
- 符合支付行業安全標準（PCI DSS）

## 系統邊界

### 內部系統
- 使用者管理系統
- 捐款管理系統
- 支付整合系統
- 管理後台系統

### 外部系統
- 第三方支付閘道（Stripe、藍新金流）
- 郵政系統（郵政劃撥單處理）
- 雲端服務（AWS/GCP/Azure）
- 電子郵件服務（通知郵件發送）

## 假設與限制

### 假設
1. 使用者擁有有效的電子郵件地址
2. 使用者擁有信用卡或郵政劃撥帳號
3. 網路連線穩定
4. 第三方支付閘道服務正常運作

### 限制
1. 捐款金額上限：單筆 100 萬元（可設定）
2. 捐款取消時間限制：24 小時內
3. 郵政劃撥單處理時間：3-5 個工作天
4. 信用卡分期付款僅支援特定銀行
